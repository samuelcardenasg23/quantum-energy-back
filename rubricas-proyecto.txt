1. Descripción General
Como parte del cierre del curso de Sistemas Distribuidos, cada equipo desarrollará un proyecto práctico integral que demuestre la capacidad de aplicar los conceptos estudiados, con énfasis en el diseño, despliegue y operación de una solución realista sobre una nube pública (AWS, Azure, GCP o Oracle Cloud – OCI).
El proyecto debe evidenciar el dominio de herramientas de automatización, capacidades de integración entre componentes distribuidos y pruebas técnicas relacionadas con al menos un eje del modelo PAM:

P (Performance)
A (Availability)
M (Management)

2. Objetivos

Aplicar conocimientos técnicos en un entorno real de infraestructura distribuida.
Usar herramientas y servicios de nube pública para construir soluciones escalables.
Integrar distintos servicios, aplicaciones o capas tecnológicas de manera funcional.
Evaluar la solución desde una dimensión técnica (performance, disponibilidad o gestión).
Preparar y presentar una sustentación técnica coherente, clara y basada en evidencia.

3. Composición del equipo

Equipos de 4 integrantes.
Todos los miembros deben participar activamente en diseño, implementación y presentación.
El liderazgo técnico y la distribución de roles deberá quedar registrada en el entregable.

4. Requisitos del Proyecto
4.1. Tema Base
Desarrollar un sistema funcional distribuido en una nube pública que:

Consuma e integre al menos dos componentes distribuidos (por ejemplo: frontend + backend, base de datos + motor de eventos, almacenamiento + analítica, etc.).
Implemente infraestructura como código para el despliegue del entorno.
Pruebe y documente uno de los tres ejes PAM: rendimiento, disponibilidad o gestión.

4.2. Componentes obligatorios

Uso de nube pública: el sistema debe estar desplegado en al menos un proveedor (AWS, Azure, GCP, OCI).
Automatización: uso de herramientas como Terraform, Ansible, Helm, etc.
Integración entre componentes: vía APIs, eventos, colas de mensajería, bases de datos compartidas, etc.
Evaluación técnica de PAM:

Performance: pruebas de carga, latencia, tiempo de respuesta, escalamiento.
Availability: despliegue activo-activo/pasivo, clustering, failover, replicación.
Management: observabilidad, trazabilidad, paneles de monitoreo, alertas.


Sustentación técnica: presentación con demo funcional y justificación de decisiones.
